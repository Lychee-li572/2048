# 2048 Mini Game Design

## Goal

Build a minimal, framework-free 2048 game that runs as a static web page. The game should be playable with the keyboard arrow keys, support restarting, show the current score, and persist a top-10 high-score list in the browser.

## Scope

- Single static page with `index.html`, `style.css`, and `script.js`
- 4x4 board
- Keyboard controls: Up, Down, Left, Right
- Merge behavior consistent with standard 2048 rules
- Current score display
- Restart button
- Game-over modal overlay showing the final score
- Local high-score ranking stored in `localStorage`

## Layout

The page stays intentionally simple:

- Title
- Score panel with current score
- Restart button
- 4x4 board
- Small leaderboard section showing top 10 scores
- Hidden modal overlay that appears on game over

The visual style should be clean and minimal, using plain blocks and strong contrast so the board is easy to read.

## Game Logic

The board state is represented as a 4x4 number grid.

On game start:

- Reset score to 0
- Create an empty 4x4 board
- Spawn two starting tiles

On each valid move:

- Compress tiles toward the move direction
- Merge equal adjacent tiles once per move
- Increase the score by the merged tile values
- Spawn one new tile in a random empty cell
- Re-render board and score

If a move does not change the board, no new tile is spawned.

## End Game

The game is over when:

- No empty cells remain
- No horizontal or vertical merges are possible

When game over happens:

- Save the final score into local leaderboard history
- Sort scores descending
- Keep only the top 10 entries
- Persist the result to `localStorage`
- Show a centered modal overlay with the final score and a restart button

## Data Persistence

Use one `localStorage` key for leaderboard data.

Stored shape:

- Array of score records
- Each record includes score and timestamp

The UI only needs to display ranking order and score. Timestamp exists mainly to make records stable if scores tie.

## Error Handling

- If `localStorage` is unavailable or invalid, fall back to an empty leaderboard
- If stored data cannot be parsed, ignore it and recreate a clean list

## Testing

Manual verification is enough for this project:

- Arrow keys move tiles in all four directions
- Merges only happen once per move
- Score increases correctly after merges
- Restart fully resets the board and score
- Game-over modal appears correctly
- Final score is recorded
- Leaderboard persists after page refresh
- Only the top 10 scores are kept
